# Інструкції по імпорту тест-кейсів в TestRail

## Файл для імпорту
**PUMB_Check_Information_TestRail_Import.csv** - містить 15 тест-кейсів для сторінки ознайомлення з інформацією ПУМБ

## Кроки імпорту в TestRail

### 1. Підготовка до імпорту
1. Увійдіть в TestRail під обліковим записом з правами адміністратора
2. Створіть новий проект або виберіть існуючий проект "ПУМБ Бізнес"
3. Створіть нову секцію "Реєстрація ФОП - Ознайомлення з інформацією"

### 2. Імпорт CSV файлу
1. Перейдіть в розділ **Test Cases**
2. Виберіть проект та секцію
3. Натисніть кнопку **"Add Test Case"** → **"Import Test Cases"**
4. Виберіть файл **PUMB_Check_Information_TestRail_Import.csv**
5. Налаштуйте маппінг полів:
   - **Title** → Title
   - **Type** → Type
   - **Priority** → Priority
   - **Estimate** → Estimate
   - **Preconditions** → Preconditions
   - **Steps** → Steps
   - **Expected Result** → Expected Result
   - **Postconditions** → Postconditions
   - **References** → References
   - **Automated** → Custom Field (Automated)

### 3. Налаштування полів в TestRail

#### Обов'язкові поля:
- **Title** - назва тест-кейсу
- **Type** - тип тестування (Functional, UI, Validation)
- **Priority** - пріоритет (High, Medium, Low)
- **Estimate** - оцінка часу виконання

#### Додаткові поля:
- **Preconditions** - передумови для виконання
- **Steps** - кроки виконання тесту
- **Expected Result** - очікуваний результат
- **Postconditions** - дії після тесту
- **References** - посилання на User Story
- **Automated** - чи автоматизований тест (YES/NO)

### 4. Налаштування кастомних полів

Якщо в TestRail відсутні деякі поля, створіть їх:

1. Перейдіть в **Administration** → **Custom Fields**
2. Створіть нові поля:
   - **Automated** (Dropdown: YES, NO)
   - **References** (Text field)
   - **Postconditions** (Text field)

### 5. Перевірка імпорту

Після імпорту перевірте:
1. Всі 15 тест-кейсів імпортовані
2. Поля заповнені коректно
3. Пріоритети встановлені правильно
4. Типи тестування призначені
