# Тест-сьют: Панель тегів - SmarterHumans Flashcards

## Опис тест-сьюту
Цей тест-сьют покриває функціонал панелі тегів у flashcard додатку SmarterHumans, включаючи створення, редагування, видалення тегів, фільтрацію карток та UI тестування.

## Загальна статистика
- **Загальна кількість тест-кейсів:** 20
- **Позитивні тест-кейси:** 6
- **Негативні тест-кейси:** 4
- **UI тест-кейси:** 3
- **Тест-кейси валідації:** 2
- **Edge-кейси:** 3
- **Тест-кейси продуктивності:** 2

## Розподіл за пріоритетами
- **Critical:** 0
- **High:** 6
- **Medium:** 8
- **Low:** 6

## Розподіл за типами
- **Functional:** 9
- **UI:** 3
- **Validation:** 2
- **Other:** 2

## Список тест-кейсів

### Позитивні тест-кейси
1. **TC_TAGS_001** - Створення нового тегу (High Priority)
2. **TC_TAGS_002** - Додавання тегу до картки (High Priority)
3. **TC_TAGS_003** - Фільтрація карток за тегом (High Priority)
4. **TC_TAGS_004** - Редагування назви тегу (Medium Priority)
5. **TC_TAGS_005** - Видалення тегу (Medium Priority)
6. **TC_TAGS_006** - Пошук тегів (Medium Priority)

### Негативні тест-кейси
7. **TC_TAGS_007** - Створення тегу з порожньою назвою (High Priority)
8. **TC_TAGS_008** - Створення тегу з дублікатом назви (High Priority)
9. **TC_TAGS_009** - Створення тегу з занадто довгою назвою (Medium Priority)
10. **TC_TAGS_010** - Видалення тегу, який використовується в картках (Medium Priority)

### UI тест-кейси
11. **TC_TAGS_011** - Відображення панелі тегів (High Priority)
12. **TC_TAGS_012** - Адаптивність панелі тегів (Medium Priority)
13. **TC_TAGS_013** - Візуальний стан активного тегу (Medium Priority)

### Тест-кейси валідації
14. **TC_TAGS_014** - Валідація спеціальних символів у назві тегу (Medium Priority)
15. **TC_TAGS_015** - Валідація пошуку тегів (Low Priority)

### Edge-кейси
16. **TC_TAGS_016** - Створення тегу з мінімальною довжиною (Low Priority)
17. **TC_TAGS_017** - Створення тегу з максимальною довжиною (Low Priority)
18. **TC_TAGS_018** - Фільтрація при відсутності карток з тегом (Low Priority)

### Тест-кейси продуктивності
19. **TC_TAGS_019** - Завантаження панелі тегів з великою кількістю тегів (Medium Priority)
20. **TC_TAGS_020** - Пошук серед великої кількості тегів (Medium Priority)

## Загальний час виконання
**Орієнтовний час виконання всього тест-сьюту:** 58 хвилин

## Передумови для виконання тестів
- Доступ до додатку SmarterHumans Flashcards
- Авторизований користувач з правами на роботу з тегами
- Тестові дані (картки, теги)

## Залежності між тест-кейсами
- TC_TAGS_002 залежить від TC_TAGS_001 (потрібен створений тег)
- TC_TAGS_003 залежить від TC_TAGS_002 (потрібні картки з тегами)
- TC_TAGS_004, TC_TAGS_005 залежать від TC_TAGS_001 (потрібен створений тег)
- TC_TAGS_010 залежить від TC_TAGS_002 (потрібен тег, прив'язаний до карток)

## Рекомендації по виконанню
1. Спочатку виконати позитивні тест-кейси високого пріоритету
2. Потім негативні тест-кейси валідації
3. UI тест-кейси можна виконувати паралельно з функціональними
4. Edge-кейси та тести продуктивності виконувати в останню чергу

## Автоматизація
Всі тест-кейси позначені як "NO" для автоматизації. Рекомендується розглянути можливість автоматизації для:
- TC_TAGS_001, TC_TAGS_002, TC_TAGS_003 (основна функціональність)
- TC_TAGS_007, TC_TAGS_008, TC_TAGS_009 (валідація)
- TC_TAGS_011 (UI тестування)
